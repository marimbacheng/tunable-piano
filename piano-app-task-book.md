# 可調基準音（A4）練唱鋼琴 App — 實作任務書

> 交付給 Claude Code 逐里程碑實作。每個 Milestone 完成後須通過「驗收條件」才進入下一關。

---

## 0. 專案目標

在 iPhone（橫幅）上運行的網頁鋼琴，讓使用者依原曲標準音（A4=440 / 432 / …）平移整組琴鍵音高，主要輔助**練唱抓音**。音源為乾淨的合成持續音（非擬真鋼琴取樣）。

---

## 1. 交付形式與技術棧

- **開發期**：多檔模組化（見「檔案結構」）。純前端，無 build step。
- **收尾（M6，選做）**：可選擇性 inline 打包成單一 `index.html` 方便手機測試/分享。
- **技術棧**：Vanilla JS + [Tone.js](https://tonejs.github.io/)（via CDN）+ 純 CSS。**不使用**任何前端框架。
- **CDN**：Tone.js 由 CDN 載入（開發期）；若 M6 要離線，再改為內嵌或 service worker 快取。
- **目標平台**：iOS Safari 15+（主目標）、桌機 Chrome/Safari（開發期）。RWD。
- **部署目標**：**GitHub Pages**（純靜態、免費、自動 HTTPS，貼合既有 GitHub 工作流）。
- **資產一律相對路徑**：因 Pages project site 為 `username.github.io/repo/` 子路徑，所有 css/js 引用一律用相對路徑（如 `css/style.css`、`js/main.js`），**禁用**絕對路徑（`/css/...`），以免子路徑下 404，並讓專案與 host 解耦（將來換 Vercel/Netlify 免改）。

---

## 2. 目標平台與硬性限制（寫死，不要另尋做法）

1. **AudioContext 必須由使用者手勢啟動**：開場需一個「點擊開始」解鎖畫面，於 `pointerdown`/`click` 內呼叫 `Tone.start()`。未解鎖前不得嘗試發聲。
2. **橫向無法強制**：iOS 對 manifest `orientation` 支援差，不要試圖鎖定。改為 RWD + 偵測到**直向**時全螢幕蓋一層「請將手機橫放」提示，橫向時自動移除。
3. **觸控**：發聲一律用 `pointerdown`（非 `click`，避免延遲）。全域 `touch-action: none`、`user-select: none`，阻止雙擊縮放與橡皮筋捲動。同時支援滑鼠（桌機開發）。
4. **節拍器排程**：**禁止**用 `setInterval` 直接發聲。使用 `Tone.Transport` / audio clock 排程，`setInterval` 僅可用於前瞻排程迴圈。
5. **安全區**：瀏海/動態島用 `env(safe-area-inset-*)` 處理。

## 3. 核心規格（已定案）

### 3.1 音訊引擎
- **音源**：乾淨合成**持續音**。用 `Tone.PolySynth`（需多鍵同響），振盪器建議 `triangle` 或 sine 加少量泛音，使音高清楚可辨、不刺耳。
- **音高公式（等律，僅平移基準）**：
  ```
  f = A4ref × 2^((midi − 69) / 12)
  ```
  其中 A4ref 為使用者設定的 A4 頻率，midi 為該鍵之 MIDI 音高。**律制固定為等律**，A4 只做基準平移。
- **按鍵包絡**：點一下＝`triggerAttackRelease`，觸發固定約 2 秒的衰減，讓聲音在 ~2 秒內完全漸弱。attack 短（~5–15ms），release 讓尾音在 2 秒內歸零。**先不加延音/按住開關**。
- **主音量**：全域 `Tone.Destination` 或一個主 Gain，UI 可即時調整。

### 3.2 A4 基準音設定
- UI：數字輸入 + 加/減符號，**步進 1 Hz**，**範圍 415–445 Hz**（超界鉗制）。
- 常見值可作為顯示參考（440/442/432/415），但主控制是數字輸入 + 加減。
- 改變 A4 後，整組鍵盤即時套用新音高（不需重載）。

### 3.3 鍵盤
- **音域**：A1（MIDI 33）到 C6（MIDI 84），共 52 鍵（含黑白鍵）。
- **資料模型**：先建立完整 A1–C6 鍵陣列（每鍵含 midi、白/黑、音名），渲染只顯示「可視視窗」的子集。
- **可視鍵數**：預設 **12**，最小 **6**，最大 **20**，由 +/− 增減（超界鉗制）。**鍵少 → 鍵變寬**（好按）；鍵盤寬度填滿容器，單鍵寬 = 容器寬 / 可視鍵數。
- **上方卷軸**：一條可拖動的小卷軸，移動的是「可視視窗在 A1–C6 範圍內的位置」。視窗不得超出音域邊界。
- **白鍵標記**：科學音高記號，如 `C4`、`D4`、`A4`…（含八度數字）。中央 C = C4 = MIDI 60；A4 = MIDI 69（＝基準音）。**黑鍵不標音名**。
- **按下視覺**：琴鍵顏色變深；**放開即恢復**（與聲音的 2 秒衰減互不相關，放開就恢復顏色）。

### 3.4 節拍器（左上角）
- **排程**：`Tone.Transport`，非 setInterval 發聲。BPM 準確。
- **BPM 範圍**：20–400。
- **拍號**：可設定；用於視覺拍點顯示與未來重音擴充。**目前無重音**（所有拍同音量）。
- **Tap 設速**：一個方塊，上面顯示「tap」字樣，點擊處即為 tap 區。取**最近 2 下**的間隔換算 BPM。（可自訂：超過一定時間未 tap 則重置暫存。）
- 節拍音：短促 click（例如短包絡的合成 blip / `Tone.MembraneSynth`），與琴鍵音源分離。

### 3.5 Drone（持續參考音）
- 定調後可將某音**長按或鎖定成持續音**，方便跟著唱。
- 用**獨立**於 3.1 一般按鍵的機制實作（持續 attack、不套 2 秒 release），避免與點按音衝突。可再次操作解除。

### 3.6 狀態保存
- `localStorage` 記住：A4、BPM、拍號、可視鍵數、可視視窗位置、主音量。重開沿用。

## 4. 檔案結構（開發期）

```
/
├── index.html          # 結構、解鎖畫面、橫向提示層、CDN 載入 Tone.js
├── css/
│   └── style.css       # RWD、safe-area、鍵盤/控制列樣式、按下態
└── js/
    ├── audio.js        # 音高公式、PolySynth、triggerAttackRelease、drone、主音量
    ├── keyboard.js     # A1–C6 資料模型、可視視窗、鍵寬計算、渲染、點按與變色
    ├── metronome.js    # Tone.Transport、BPM、拍號、tap 取速
    ├── ui.js           # A4/BPM/鍵數 控制列、卷軸、localStorage 存取
    └── main.js         # 手勢解鎖、橫向偵測、模組組裝與事件綁定
```

M6 若打包：另產出 `dist/index.html`（inline 全部）。

## 5. Milestones 與驗收條件

### M0 — 骨架 / 解鎖 / 橫向偵測
- 建立檔案結構、載入 Tone.js。
- 「點擊開始」解鎖畫面 → 手勢內 `Tone.start()`。
- 直向顯示「請將手機橫放」全螢幕提示，橫向自動移除。
- 全域 `touch-action: none` / `user-select: none` / safe-area。
- **驗收**：桌機與手機開啟，點擊解鎖後進入（空的）主畫面；旋轉裝置時提示層正確切換。

### M1 — 音訊引擎 + A4 可調
- 實作音高公式與 `PolySynth`。`playNote(midi)` 發乾淨持續音，~2 秒內完全衰減；支援多鍵同響。
- A4 數字輸入 + 加減（415–445、步進 1）。
- **驗收**：呼叫 `playNote(69)` 在 A4=440 時實測 ≈440.0 Hz、A4=432 時 ≈432.0 Hz，誤差 < 0.5 Hz；`playNote(60)`（C4）在 440 制 ≈261.63 Hz、432 制 ≈256.87 Hz。多鍵同響不爆音。

### M2 — 鍵盤渲染 + 音名
- A1–C6 資料模型；渲染預設 12 鍵可視視窗；白鍵標 C4/D4… 含八度數字，黑鍵不標。
- 點鍵：發正確音高、顏色變深、放開恢復（`pointerdown`）。
- **驗收**：畫面顯示 12 鍵，白鍵音名正確、黑鍵無字；任一鍵點下發對應音高、變色，放開即復原。

### M3 — 鍵數增減 + 卷軸
- +/− 改可視鍵數（6–20 鉗制）；鍵少→鍵變寬（填滿容器）。
- 上方卷軸移動可視視窗於 A1–C6；邊界不出界。
- **驗收**：+/− 正確改變鍵數且鉗制；卷軸可平移涵蓋 A1–C6 全域；最左/最右不越界；鍵寬隨鍵數即時變化。

### M4 — 節拍器
- `Tone.Transport` 排程；BPM 20–400；拍號可設；無重音；tap 取最近 2 下平均。
- 「tap」方塊。
- **驗收**：設 BPM=120 播放，間隔誤差可聽感穩定（audio clock 排程，非 setInterval）；tap 兩下算出對應 BPM；換拍號時視覺拍點循環正確；背景/節流下不明顯飄拍。

### M5 — 音量 + Drone + 狀態保存
- 主音量即時控制。
- 長按/鎖定某音為 drone；可解除。
- `localStorage` 保存並在重開沿用（A4、BPM、拍號、鍵數、視窗位置、音量）。
- **驗收**：拖主音量即時生效；drone 可持續發聲並可解除、不受 2 秒衰減影響；重開後上述設定保留。

### M6（選做）— PWA / 離線 / 單檔打包 + 手機微調
- 視需求加 manifest + apple meta（standalone/全螢幕）、service worker 快取。
- **離線關鍵**：一旦要離線 PWA，**必須把 Tone.js 從 CDN 改為自帶**（下載自 host 或內嵌），否則斷網時載入失敗。service worker 一併快取本地資產。
- GitHub Pages 自帶 HTTPS，service worker 與「加到主畫面」全螢幕皆可正常運作（SW 強制需 HTTPS）。
- 打包 `dist/index.html`。
- 於實機微調觸感（鍵寬、按下延遲、卷軸手感）。
- **驗收**：加到主畫面可全螢幕啟動；（若做離線）飛航模式仍可用；實機練唱手感確認。
